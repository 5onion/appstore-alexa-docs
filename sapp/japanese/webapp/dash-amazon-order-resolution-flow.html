Amazon 発注解決手順 (DRS)

概要

デバイスメーカーが製品の再発注にReplenishment API を使用する際、エラーが生じて Amazon への発注処理が遂行できないことがあります。 発注時によくある問題には、住所に誤りがある場合、支払い方法が不正確な場合、再発注された製品が Amazon で一時的に品切れとなっている場合などがあります。 こういった場合、Amazon はお客様に問題の解決を促し、適切なリソースを提示します。 デバイスメーカーとして、お客様への対応には Amazon の発注解決手順を理解していることが重要です。

正常な発注手順

次に挙げるのは、発注処理に問題がない正常な手順の一例です。

  1. DRS がデバイスメーカーのReplenishment API リクエストを受け取る。
  2. デバイスメーカーは「200 OK」（StandardOrderPlaced）レスポンスを受け取る。
  3. デバイスメーカーは（OrderPlaced）の SNS 通知を受け取る。
  4. お客様は注文が作成された旨のEメールを受け取る。
  5. お客様は注文の取り消しまたは変更がある場合、24時間以内に行う。
  6. これ以降のReplenishment API コールは商品が配送されるまで、「200 OK」（OrderInProgress）レスポンスを受け取る。

Amazon 発注解決手順

次に挙げるのは、注文の処理の際に問題が生じた手順の例です。

  1. DRS がデバイスメーカーのReplenishment API リクエストを受け取る。
  2. デバイスメーカーは「200 OK」（StandardOrderPlaced）レスポンスを受け取る。
  3. デバイスメーカーが（OrderPlaced）の SNS 通知を受け取らない。
  4. お客様が Amazon 発注解決のEメールを受け取る。
  5. お客様はこれより72時間以内に問題解決しなければならない。
  6. これ以降のReplenishment API コールは「200 OK」（OrderInProgress）レスポンスを受け取る。

シナリオ1： お客様が72時間以内に問題解決しない

  1. お客様は注文が取り消された旨のEメールを受け取る。
  2. デバイスメーカーは（OrderCancelled）の SNS 通知を受け取る。
  3. デバイスメーカーのReplenishment API コールはブロックを解除され、新規の「200 OK」（StandardOrderPlaced）を受け取る。

シナリオ2： お客様が72時間以内に問題を解決する

  1. お客様は注文が作成された旨のEメールを受け取る。
  2. デバイスメーカーは（OrderPlaced）の SNS 通知を受け取る。
  3. お客様は注文を取り消す場合、24時間以内に行う。
  4. これ以降のReplenishment API コールは「200 OK」（OrderInProgress）レスポンスを受け取る。
