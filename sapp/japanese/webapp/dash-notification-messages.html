SNS 通知メッセージ (DRS)

概要

Amazon はデバイス変更、オーダー状況、サービス加入の状況、その他多様な通知を Amazon の簡易通知サービス （SNS、Simple Notification Service）で送信します。 Amazon SNS により、アプリケーションはスピードが重視されるメッセージを「プッシュ」メカニズムを通して複数の購読者に向けて配信することができます。こうして定期的に更新のチェックをしなくても済みます。 例えば、Dash Replenishmentシステムサービス（DRS、Dash Replenishment Service）はデバイスの登録解除やイベントの不具合を貴殿/貴社（デバイスメーカー）に通知するのに SNS を使用し、配送状況の提供に通知を使用します。

以下の項目では、通知のサンプルが参考用に JSON で提供されます。

Note: 通知には将来新しい項目が追加されることがありますが、貴殿/貴社のサービスコードは不要な項目に対処し無視しなくてはなりません。

DRS では以下の通知をサポートしています。

Tip: SNS は必ずしもメッセージを連続的に受け取るものではありません。 非常に狭い時間枠（30-60秒）でテストを行う場合はこのことを念頭に置いてください。 SNS はメッセージを受信した順番に送信しますが、メッセージが「送信」された順番に受信したことを保証するものではありません。 SNS メッセージペイロード内の通知時刻欄は DRS がメッセージを送信した時刻を示します。

発注がありました

受注確認の通知は DRS がReplenishment API コールを受け取り、注文に何の問題もなかった場合に送信されます。 このメッセージを受信次第、デバイスメーカーはReplenishment API リクエストから受け取った注文が、24時間以内に Amazon のお客様により取り消されない場合、発送されることが通知されます。

注文が取り消されました

注文の取り消し通知の送信には2つのシナリオがあります。

  1. 正常に発信された注文がお客様の手で取り消されたとき。

  2. Replenishment API リクエストが Amazon オーダー解決手順を引き起こし、Amazon のお客様が72時間以内に問題を解決できないとき。

以下は参考用に、注文の取り消しの通知のサンプルです。

            "modelId": "myDeviceModel"

デバイスが登録されました

デバイスの登録通知はお客様が LWA を正常に完了したときに随時送信されます。 通知はお客様がいずれかの ASIN を選ぶ前に送信されます。

以下は参考用に、デバイスの登録通知のサンプルです。

{
    "notificationInfo": {
        "notificationType": "DeviceRegisteredNotification", "LWAClientId": "amzn1.application-oa2-client.6b68e3c7ba3c434baa9283ad8747d469", "notificationTime": "2015-10-01T02:23:52.747Z", "notificationId": "amzn1.dash.notification.v1.4eab8f47-f876-4650-a74a-5ca90bb657c0", "version": "2015-06-05"
    },
    "deviceInfo": {
        "deviceIdentifier": {
            "serialNumber": "mySerial"
        },
        "productIdentifier": {
            "modelId": "myDeviceModel"
        }
    },
    "customerInfo": {
        "directedCustomerId": "amzn1.account.AEYAPUSDAKRB7KDV5WSAAIAIVMSA"
    }
}

デバイスが登録解除されました

デバイスの登録解除通知の送信には多数の理由があります。以下に例を挙げます。

  1. デバイスメーカーが登録解除 API をコールする。

  2. Amazon のお客様が Amazon.com に行き、デバイスを何らかの Amazon.com 管理ページから登録解除する。

  3. Amazon のお客様が Amazon カスタマーサービスに連絡し、カスタマーサービスの係員がお客様に代わってデバイスを登録解除する。

  4. Amazon のお客様が「Amazonアカウントによる外部サイトへのログインの設定 」に行き、デバイスメーカーとの関係を削除する。 これによりデバイスメーカーと関連したあらゆるお客様のデバイスが登録解除されます。

以下は参考用に、受注確認の通知のサンプルです。

{
    "notificationInfo": {
        "notificationType": "DeviceDeregisteredNotification", "LWAClientId": "amzn1.application-oa2-client.6b68e3c7ba3c434baa9283ad8747d469", "notificationTime": "2015-09-29T18:03:28.669Z", "notificationId": "amzn1.dash.notification.v1.3b060d60-f5e1-49d1-93f9-4020f0060297", "version": "2015-06-05"
    },
    "deviceInfo": {
        "deviceIdentifier": {
            "serialNumber": "mySerial"
        },
        "productIdentifier": {
            "modelId": "myDeviceModel"
        }
    },
    "customerInfo": {
        "directedCustomerId": "amzn1.account.AFOM77AMO524CJY3HPKB2GIJCBAQ"
    }
}

サービス加入が変更されました

サービス加入変更通知の送信には4つのシナリオがあります。

  • お客様が新規の LWA デバイス登録を完了したとき
  • お客様が以前選択のなかった ASIN を選択したとき
  • お客様がこれまで選択されていた ASIN の選択を解除したとき
  • お客様がサービス加入のスイッチのオン・オフを切り替えたとき

以下は参考用に、サービス加入の変更通知のサンプルです。

{
    "notificationInfo": {
        "notificationType": "SubscriptionChangedNotification", "LWAClientId": "amzn1.application-oa2-client.1bf9d3fa83bc4c50968ecf595ace1127", "notificationTime": "2015-09-30T23:27:50.771Z", "notificationId": "amzn1.dash.notification.v1.067cb532-ec2d-4ab4-a0ed-6e3da599cb83", "version": "2015-06-05"
    },
    "deviceInfo": {
        "deviceIdentifier": {
            "serialNumber": "mySerial"
        },
        "productIdentifier": {
            "myDeviceModel": "DrsBountySampleDevice"
        }
    },
    "subscriptionInfo": {
        "slotsSubscriptionStatus": {
            "vacuumBag": true, "vacuumFilter": true
        }
    },
    "customerInfo": {
        "directedCustomerId": "amzn1.account.AEZPSLGB77YDGN55ESXOP2WWA3ZQ"
    }
}
