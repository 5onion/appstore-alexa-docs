Dash Replenishment Service (DRS)

概要

Amazon Dash Replenishment Service（DRS、Dash 補充サービス）では消耗品が少なくなってきた時、例えば Brita の浄水器のフィルターの追加注文のように、ネットに接続したデバイスが物理的商品を Amazon に注文することができます。 Amazon Dash Replenishment serviceの使用により、デバイスメーカーはお客様に Amazon の低価格、幅広いチョイス、信頼性の高い配送を提供しつつ、Amazon の認証および決済システム、カスタマーサービスおよびフルフィルメントのためのネットワークを活用することができます。

DRS セルフサービスポータル

DRS セルフサービスポータルは DRS に紐付いたデバイスの作成・管理をサポートします。 デバイスウィザードは DRS の事前構成プロセスを簡易化し、デバイスモデルの作成および製品 ASIN のリストの設定をガイドします。これにより Amazon Dash Replenishment Serviceのテストと統合が開始できます。 ここをクリックして開始してください。

認証

DRS を利用するには、貴社の製品は LWA アクセストークンを取得する必要があります。これにより貴社の製品はお客様に代わって API へのアクセスが付与されます。 DRS で利用できるようにするには、2通りのアクセストークンの取得方式があります。

  • LWA ウェブ API 方式

  • LWA モバイル SDK 方式

ベース URL

DRS はあらゆるエンドポイントをビルトオンした2つのベース URL をサポートします。

サービスコール

例えば注文のReplenishmentやデバイスのスロット状況の報告など、サービス関連のあらゆる API コールには次のベース URL を使用しなければなりません。

https://dash-replenishment-service-na.amazon.com

ウェブ関連のコール

あらゆるウェブ関連のコールには次のベース URL を使用しなければなりません。 貴殿/貴社の設定およびティザーページなど、対顧客のページおよびインタラクションにこの URL をご使用ください。

https://drs-web.amazon.com

API バージョニング

Amazon では API の機能性を、お客様に直接影響することなく漸次更新するため、バージョニングを使用しています。 あらゆる API コールには次のヘッダを指定することが必須です： x-amzn-type-version および x-amzn-accept-type。

  • x-amzn-type-version – 貴殿/貴社は送信するリクエストのバージョンを制御できます。
  • x-amzn-accept-type – 貴殿/貴社は受信するレスポンスのバージョンを制御できます。

サポートするバージョンは各 DRS エンドポイントに対応します。
