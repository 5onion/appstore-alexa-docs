SNS トピックを作成 (DRS)

概要

Dash Replenishment Service（DRS、補充サービス）は SNS トピックを使用してデバイスの状況、発注状況、サービス加入状況などの変更を貴殿/貴社（デバイスメーカー）に通知します。 Amazon の簡易通知サービス（SNS、Simple Notification Service）は設定、管理および、アプリケーションやユーザーへのメッセージの即時送信を簡易にする AWS による出版-購読型サービスです。 サポートしている DRS 通知を網羅したリストは通知メッセージをご覧ください。

SNS トピックの作成方法は以下の説明をご覧ください。

DRS 用 SNS トピックを作成する

  1. AWS コンソール にログインします。
  2. 「モバイルサービス」から SNS を選んでください
  3. 左側のナビゲーションからTopicsをクリックし、次にCreate new topicをクリックします。
  4. トピックおよび表示名を入力し、その後「Create topic」をクリックします。
    • Topic nameは貴殿/貴社の SNS 通知を設定するのに使用される Amazon リソース名（ARN、Amazon Resource Name）と呼ばれる固有の永続的識別番号の作成に使用されます。 注意： 親しみやすいトピック名および表示名を選び、コードの可読性を高めましょう。

貴殿/貴社のトピックへのメッセージの発行を DRS に許可する

  1. 貴殿/貴社が作成したトピックの検索には ARN をクリック。
  2. Other topic actionsをクリックし、Edit topic policyをクリック。
  3. Basic viewから、Allow these users to publish messages to this topicの下で、Only these AWS user を選択し、貴殿/貴社の AWS アカウント ID および DRS AWS アカウント ID を入力してください。
    • AWS コンソールに移動し、貴殿/貴社の AWS アカウント ID にアクセスする。
    • DRS AWS アカウント ID に以下を入力します。 476877469412
  4. 方針を更新する をクリック。
