LWA ウェブ API の使用 (DRS)

概要

Dash Replenishment Service（DRS、Dash 補充サービス） API にアクセスするには、DRS を有効化した貴殿/貴社の製品が、API へのアクセス権を付与する「Amazon にログイン （LWA、Login With Amazon）」アクセストークンを製品のユーザーに代わって取得する必要があります。

本書はコンパニオン製品登録ウェブサイトを有する製造業者を対象としています。

認証タイプ

LWA は2種の認可タイプをサポートしていますが、DRS は認可コードの付与を必要とします。

Amazon にログイン同意リクエストを作成

同意リクエストの作成により、貴殿/貴社のユーザーは Amazon にログイン（LWA、Login with Amazon）ウェブサイトにリダイレクトされ、そこで貴殿/貴社の製品で特定のインスタンスが DRS で使用できるように、ユーザー自身の Amazon.com 認証情報を入力します。

同意リクエストは貴殿/貴社の製品でユーザーに特定のインスタンスに関する情報を含みます。 貴殿/貴社の製品の登録ウェブサイトはこの情報を収集できます。情報にはシリアル番号や MAC アドレスなど、製品タイプ ID および固有の製品識別番号が含まれます。 製品タイプ ID は貴殿/貴社が製品を登録する際に Amazon デベロッパーポータルに設定するものです。

同意リクエストは以下の URL 符号化パラメータとともにユーザーを LWA（https://www.amazon.com/ap/oa）にリダイレクトすることにより構築されます。

  • client_id: ウェブサイトのクライアント ID。 この情報は Amazon のデベロッパーポータル上のAmazon にログインのページでご覧いただけます。
  • scope: DRS 内の許可レベルは次のひとつです。dash:replenish

  • scope_data: 製品情報を含む補足的 JSON データ。 この JSON のシンタックスは以下の通りです。

  • device_model: この製品タイプに指定された製品タイプ ID。 製品タイプ ID は Amazon デベロッパーポータルでの製品登録プロセスの一環で指定済みです。

  • serial: 製品のインスタンスを識別する固有のキー。 これは例えばシリアル番号であったり MAC アドレスであったりします。

  • response_type: 認可コード付与用 code。
  • redirect_uri: 貴殿/貴社がサインアップ時にアプリのセキュリティプロフィールに追加した返信 URI のひとつを特定。

リフレッシュおよびアクセストークンを取得

ユーザーが認証されると、ユーザーは貴殿/貴社がリクエストの redirect_uri パラメータ内で提供した URI にリダイレクトされます。

認可コードの付与

LWA からのレスポンスは認可コードを含みます。 貴殿/貴社が認可コードを受信すると、最初の LWA 登録が正常に完了したことを意味します。 貴殿/貴社のアプリケーションはアクセストークンおよびリフレッシュトークンのために認可コードを交換する必要があります。

認可コード付与レスポンスのサンプル

https://localhost/?code=ANdNAVhyhqirUelHGEHA&scope=dash:replenish

次に、貴殿/貴社のサービスは返信された認可コードを活用してアクセストークンをリクエストします。

  • POST リクエストを以下のパラメータと共に https://api.amazon.com/auth/o2/token に送ります。

HTTP ヘッダーパラメータ

HTTP ボディパラメータ

  • grant_type: authorization_code
  • code: レスポンス内に返信された認可コード。
  • client_id: ウェブサイトのクライアント ID。 この情報は Amazon デベロッパーポータルの「Amazon にログイン」のページでご覧いただけます。
  • client_secret: ウェブサイトのクライアントの秘密。 この情報は Amazon デベロッパーポータルの「Amazon にログイン」のページでご覧いただけます。
  • redirect_uri: redirect_uri は LWA から許可されたリンク元 URL のいずれかと一致する必要があります（これらはサインアップの際に貴殿/貴社のアプリのセキュリティプロフィールに追加されているべきものです）。
  • code_verifier: code_verifier は、認可APIに渡されたcode_challenge パラメータを生成するために使用されたプライベートキーであるランダムなストリングです。 code_verifierの生成方法に関する詳細は、SPOP ドキュメンテーションを参照してください。

DRS をコール

DRS をコールする際、リクエストのヘッダーにアクセストークン値を指定します。 具体的には、認可ヘッダーを作成し、それに値 Bearer を指定します。

新規のリフレッシュおよびアクセストークンをリクエスト

アクセストークンは1時間有効です。 アクセストークンが時間切れになったとき、またはなりつつあるとき、リフレッシュトークンを新規のアクセストークンと交換できます。

  • POST リクエストを次のパラメータと共に https://api.amazon.com/auth/o2/token に送信してください。

HTTP ヘッダーパラメータ

  • Content-Type: application/x-www-form-urlencoded

HTTP ボディパラメータ

  • grant_type: authorization_code
  • client_id: ウェブサイトのクライアント ID。 この情報は Amazon デベロッパーポータルの「Amazon にログイン」のページでご覧いただけます。
  • client_secret: ウェブサイトのクライアントの秘密。 この情報は Amazon デベロッパーポータルの「Amazon にログイン」のページでご覧いただけます。
  • redirect_uri: 貴殿/貴社がサインアップ時にアプリのセキュリティプロフィールに追加した返信 URI のひとつ。
  • code_verifier: code_verifier は、認可APIに渡されたcode_challenge パラメータを生成するために使用されたプライベートキーであるランダムなストリングです。 code_verifierの生成方法に関する詳細は、SPOP ドキュメンテーションを参照してください。
