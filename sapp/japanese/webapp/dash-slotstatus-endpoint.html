SlotStatus エンドポイント (DRS)

解説

スロット内容に消費があった際は必ず、デバイスが DRS にコールしスロットの現在の状況を報告しなければなりません。

パス

/slotStatus/{SLOT_ID}

HTTP 方式

POST

ヘッダーパラメータ

Authorization: Bearer (ACCESS_TOKEN)
x-amzn-accept-type: com.amazon.dash.replenishment.DrsSlotStatusResult@1.0
x-amzn-type-version: com.amazon.dash.replenishment.DrsSlotStatusInput@1.0

バージョニング

Amazon ではAPI の機能性を、お客様に直接影響することなく漸次更新するため、バージョニングを使用しています。 各コールでヘッダー x-amzn-type-version および x-amzn-accept-type を送信する必要があります。 SlotStatus エンドポイントは以下をサポートしています。

x-amzn-type-version: [com.amazon.dash.replenishment.DrsSlotStatusInput@1.0](mailto:com.amazon.dash.replenishment.DrsSlotStatusInput@1.0)

x-amzn-accept-type: com.amazon.dash.replenishment.DrsSlotStatusResult@1.0

ボディパラメータのリクエスト

リクエストボディには以下のデータを指定しなければなりません。

パラメータ 解説 タイプ
expectedReplenishmentDate 製品の補充予測日（ISO 8601形式）。 ストリング
remainingQuantityInUnit コンテナ内の残存数量（DeviceCapabilitiesGroup の作成時に提供された測定単位）。 整数
originalQuantityInUnit コンテナが満杯であった時点での製品数量（DeviceCapabilitiesGroup の作成時に提供された測定単位）。 整数
totalQuantityOnHand 手元にあるがデバイスに補充していない製品の数量（既知の場合）。 整数
lastUseDate 製品が特定のスロットから消費された直近の日時（ISO 8601形式）。 ストリング

リクエストボディのサンプル

{
    "expectedReplenishmentDate" : "2015-12-28T10:00:00Z",
    "remainingQuantityInUnit" : 3.5,
    "originalQuantityInUnit" : 10,
    "totalQuantityOnHand" : 20,
    "lastUseDate" : "2015-12-21T10:00:00Z"
}

レスポンス例

HTTP/1.1 200 OK
x-amzn-type-version: com.amazon.dash.replenishment.DrsSlotStatusResult@1.0

エラー

解説

Error

HTTP/1.1 400
Bad Request
